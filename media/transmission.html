Media Transmission

Streamed media content, in particular video, is particularly demanding for the underlying transport as it requires both enough bandwidth and low latency.

Well-deployed technologies

To ensure the user experience of streamed media remains resilient in face of changing network conditions, media providers make user of adaptive streaming which selects sources of different quality levels based on the networking environment. While several technologies are available to satisfy this approach (such as MPEG DASH), the Media Stream Extensions specification enables Web application developers to create libraries that will consume these different adaptive streaming formats and protocols.

Specifications in progress

When broadcasting popular live events on an IP network, it can be beneficial to the broadcaster to spread the load of video distribution by relying on peer-to-peer transmission among watchers. The WebRTC API enables the establishement and utilization of such a peer-to-peer connection, optimized for audio & video transport.

Exploratory work

Some devices (e.g. TV sets) provide access to non-IP broadcast media; the Tuner API in the TV Control specification brings these non-IP media streams to Web browsers.

Features not covered by ongoing work

Streaming HTTP media on HTTPs pages
While it is possible to read video/audio content served on HTTP in <audio> and <video> elements served on HTTPs pages (despite the usual mixed content restrictions), this does not extend to the usage of streaming enabled by Media Stream Extensions. There are some early discussions on how this could be solved, with documented use cases for enabling this.
